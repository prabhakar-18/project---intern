  5. 5. Do you imagine how this page would look like without HTML?
  6. 6. Looks ugly Doesnt it?
  7. 7. Without any markup to give your page content structure, the browser renders unformatted and unstyled text No HTML?
  8. 8. HTML tags give structure and meaning to your content.
  9. 9. HTML Tags <p></p> - to organize text into paragraphs <table></table> - to display table <form></form> - to define form for user input <img></img> - to add image
  10. 10. Lets add HTML tags to our page... How will it look now?
  11. 11. Looks better But still not good
  12. 12. Once page content is marked up with HTML tags, the browser applies default styles to the tags. So now page is readable and have a clear hierarchy. HTML
  13. 13. But what should we do to make the page look nice? The answer is use CSS!
  14. 14. CSS = Cascading Style Sheet
  15. 15. CSS Set of rules defining how an html element will be presented in the browser.
  16. 16. CSS Rules p { color: red; } #title { font-style: italic; border: 1px dotted blue; } .title { font-weight: bold; background: yellow; }
  17. 17. Some things you can change with CSS colors font font size backgrounds spacing sizes borders positions (layout)
  18. 18. Some things you cant change with CSS content markup
  19. 19. Lets add some css rules to our example page. So now its time for styling!!!
  20. 20. How will it look now???
  21. 21. Looks perfect!

  24. 24. JavaScript Java (c) Jeremy Keith, web developer, 2009
  25. 25. JavaScript Java JavaScript was not developed at Sun Microsystems, the home of Java. JavaScript was developed at Netscape in 1995. It was originally called LiveScript, but that name wasn't confusing enough. Java was the buzzword in the days when LiveScript was created, so LiveScript was renamed into JavaScript in the hope of catching the medias interest. But from confusion came misunderstanding
  26. 26. JavaScript Java JavaScript is NOT a Java It is NOT a light version of Java It was NOT based on Java It does NOT matter if you know Java
  27. 27. JavaScript Classic JavaScript is a client-side language used to add interactivity to your web-pages Web Browser Web Page JavaScript
  28. 28. What we can do with JavaScript? Validate values entered in the form fields
  29. 29. What we can do with JavaScript? A search box can give you suggested results while you type, based on what youve entered
  30. 30. What we can do with JavaScript? Load information automatically when it's needed
  31. 31. What we can do with JavaScript? Create photo slideshow and control how long it takes to move from one image to the next and with what animation effect
  32. 32. What we can do with JavaScript? And much much more
  33. 33. JavaScript And it does that very well! At the beginning JavaScript was designed to manipulate web pages. But nowadays its usage is much much wider: you can use it outside of web pages and the browser
